# Mini Market PRD (학습용)

## 1) 목표(무엇을 만들까?)
- 사용자가 상품을 보고
- 검색/카테고리로 골라 보고
- 장바구니에 담고
- 주문(가짜)을 저장한다

> "가짜"란? 실제 결제가 아니라, 브라우저(localStorage)에 기록만 저장하는 것.

## 2) 사용자 시나리오(사용자가 어떤 순서로 사용할까?)
1. 사이트에 들어온다.
2. 상품을 검색하거나 카테고리를 고른다.
3. 장바구니에 담는다.
4. 장바구니에서 수량을 바꾸거나 삭제한다.
5. 주문하기(가짜)를 누른다.
6. 주문 내역이 아래에 쌓인다.

## 3) MVP 기능(이번에 꼭 만들 것)
- 상품 리스트(카드 UI)
- 검색(상품명)
- 카테고리 필터
- 장바구니 담기/수량 +/- /삭제
- 주문하기(가짜) → 주문 내역 저장
- 새로고침해도 장바구니/주문 내역 유지(localStorage)

## 4) 이번엔 안 하는 것(범위 제외)
- 로그인/회원가입
- 실제 결제
- 관리자 페이지(상품 추가/삭제)

## 5) 성공 기준(완성 체크)
- 콘솔 에러 0개
- 검색/필터/장바구니/주문 기능이 동작
- 새로고침해도 장바구니/주문 내역이 남아있음
